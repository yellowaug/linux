API站点早上登录时无响应，抓包返回no response body，CDN提示回源失败
-----------------------------------------------------------------
解决方式：
IIS的预加载注意配置是否被更改，检查Web.config文件中
<applicationInitialization doAppInitAfterRestart="true" />
是否存在，不存在的话第一次加载就会特别慢，导致CDN回源失败

登录时站点提示连接服务器异常
-----------------------------------------------------------------
据开发描述是因为在更新站点时旧有的缓存仍有效导致的，解决的方法就是在IIS上重启该站点

公司平台，工厂前端站点发布时要更改配置文件
平台端：
..\mnmonline\muniumaweb\platform\config\prod.env.js
修改内容：
-----------------------------------------------------------------
module.exports = {
  NODE_ENV: '"production"',
  ENV_CONFIG: '"prod"',
  BASE_API: '"https://ptapitest.muniuma.cn"' //测试地址
  // BASE_API: '"https://ptapi.muniuma.cn"'  //线上地址
}
-----------------------------------------------------------------
..\mnmonline\muniumaweb\platform\config\dev.env.js
修改内容：
-----------------------------------------------------------------
module.exports = {
  NODE_ENV: '"development"',
  ENV_CONFIG: '"dev"',
  BASE_API: '"https://ptapitest.muniuma.cn"' //测试地址
	// BASE_API: '"https://ptapi.muniuma.cn"' //线上地址
  // BASE_API: '"http://192.168.4.54:9527"'
}
-----------------------------------------------------------------
工厂端:
..\mnmonline\muniumaweb\factory\src\utils\request.js
修改内容：
-----------------------------------------------------------------
....
// create an axios instance
const service = axios.create({
	baseURL: 'https://gcapitest.muniuma.cn', //测试地址
